# Project Glossary

## What this file is

A short list of **domain terms** whose meaning in your team differs from the industry default, or whose precise meaning matters for correctness.

Rule of thumb: if you would need to explain the word to a well-informed newcomer, it belongs here.

## Format

```
### <Term>
<Definition, one-two sentences, in your team's actual usage>
**Not to be confused with:** <the industry-default or a similar term>
**See also:** <related terms in this glossary, or external doc>
```

## Maintenance

- Add terms when an agent or colleague misunderstands one for the second time
- Remove terms that have become industry-standard
- Keep definitions short — long explanations belong in architecture notes or external docs

## Anti-patterns this file prevents

- Agents assuming industry-default meaning when your team uses the term differently
- Onboarding confusion
- Documentation drift ("I think we use X to mean Y" becoming canonical only because nobody wrote it down)
