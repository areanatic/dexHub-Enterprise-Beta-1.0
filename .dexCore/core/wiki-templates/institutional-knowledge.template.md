# Institutional Knowledge

## What this file is

A small, curated log of **load-bearing lessons** that any agent working on this project must know before making proposals. Keep the bar high — if a lesson is not worth re-stating every session, it belongs elsewhere.

## Format per lesson

```
### Lesson: <one-line headline>
- **Learned:** <YYYY-MM-DD or approximate quarter>
- **Context:** <what was being tried, one sentence>
- **Outcome:** <what happened, one sentence>
- **Rule going forward:** <what to do / not do>
- **Exception:** <when the rule does not apply>
```

## Maintenance

- Re-read quarterly. If a lesson hasn't fired in 6 months AND the conditions that produced it have changed, archive it.
- Move archived lessons to `myDex/.dex/wiki/archive/` or delete.
- If you find yourself adding a 6th lesson, ask: could two existing ones merge?

## Anti-patterns this file prevents

- Re-proposing an approach that has already failed here
- Losing the "why" behind a non-obvious convention
- Onboarding friction where a new agent has to re-discover known traps
